对，我刚刚说的就是按“后台写作台”来设计的，不是前台页面。

如果明确是后台功能，那我建议把它定义成一个完整的“文章编辑工作台”，放在后台菜单里：

## 后台模块

* 文章管理
* 新建文章
* 编辑文章
* 草稿箱
* 已发布
* 已下线
* 分类管理
* 标签管理
* 后台写文章页
## 页面建议分 3 栏：

* 顶部工具栏
* 返回列表
* 保存草稿
* 预览
* 发布文章
* 下线文章
* 显示保存状态
## 中间编辑区
* 标题输入框
* Markdown 编辑器
* 工具栏
* 图片上传并插入正文
* 右侧实时预览
## 右侧设置区
* 分类
* 标签
* 封面图上传
* 摘要
* SEO 信息
* 是否允许评论
* 是否置顶
* 是否推荐
* 发布状态
## 后台交互重点

输入后自动保存草稿
手动保存草稿
封面单独上传
正文图片上传后自动插入 Markdown
发布前校验标题、正文、slug
发布后进入“已发布”列表
编辑已发布文章时支持再次保存并重新发布
后台最核心的 4 个动作

保存草稿
上传封面
上传正文图片
发布文章
你这个后台最适合的页面结构

* 博客管理 / 文章管理
* 博客管理 / 写文章
* 博客管理 / 分类管理
* 博客管理 / 标签管理
* 博客管理 / 评论管理
* 博客管理 / 数据统计
